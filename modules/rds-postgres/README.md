# Module: `rds-postgres`

Single-instance RDS Postgres with a dedicated SG, master credentials in
Secrets Manager, custom parameter group, optional Multi-AZ, optional alarms.

## What it builds

- `aws_db_subnet_group` over the private subnets you pass in
- `aws_security_group` allowing `:5432` only from the SGs you pass in (no CIDRs)
- `aws_db_parameter_group` (Postgres `family` derived from the major version)
  with slow-query logging, connection logging, `pg_stat_statements`
- `aws_db_instance` — `gp3`, encrypted, private, parameterized
- `aws_secretsmanager_secret` (+ version) holding the master credentials JSON
- `aws_iam_role` for enhanced monitoring (60s granularity)
- 3 `aws_cloudwatch_metric_alarm` (CPU, free storage, connections) — created
  **only** if `alarm_sns_topic_arn` is set

## Secret shape

```json
{
  "username": "appadmin",
  "password": "<random>",
  "engine":   "postgres",
  "host":     "<endpoint>",
  "port":     5432,
  "dbname":   "appdb"
}
```

The application reads it at runtime — never bake it into the image or env.

## Why password is generated by the module

- Keeps secrets out of `tfvars`/state-input.
- `random_password` is in state but state is encrypted in S3+KMS (see
  `bootstrap/`).
- `lifecycle { ignore_changes = [password] }` so manual rotations via Secrets
  Manager do not get clobbered by the next `apply`.

## Inputs (selected)

| Name | Type | Default | Description |
|---|---|---|---|
| `name` | string | — | Resource prefix |
| `vpc_id` | string | — | VPC to deploy into |
| `subnet_ids` | list(string) | — | ≥ 2 private subnets |
| `ingress_security_group_ids` | list(string) | `[]` | SGs allowed on :5432 |
| `engine_version` | string | `16.4` | Postgres major.minor |
| `instance_class` | string | `db.t4g.micro` | Use `m7g.large+` for prod |
| `multi_az` | bool | `false` | Multi-AZ failover |
| `backup_retention_days` | number | `7` | 0–35 |
| `deletion_protection` | bool | `true` | Always true in prod |
| `skip_final_snapshot` | bool | `false` | dev only |
| `kms_key_arn` | string | `null` | CMK or aws/rds |
| `alarm_sns_topic_arn` | string | `null` | Skip alarms when null |

Full list in `variables.tf`.

## Outputs

| Name | Description |
|---|---|
| `endpoint` / `address` / `port` | Connection bits |
| `database_name` | Initial DB |
| `security_group_id` | Reference from app SGs to allow egress |
| `secret_arn` / `secret_name` | For `secretsmanager:GetSecretValue` policies |

## Example

```hcl
module "rds" {
  source = "../../modules/rds-postgres"

  name       = "catalog-dev-products"
  vpc_id     = module.vpc.vpc_id
  subnet_ids = module.vpc.private_subnet_ids

  ingress_security_group_ids = [module.products_api.security_group_id]

  instance_class      = "db.t4g.micro"
  multi_az            = false
  deletion_protection = false
  skip_final_snapshot = true
  alarm_sns_topic_arn = aws_sns_topic.alerts.arn
}
```

## Decisions

- **gp3 over gp2** — same baseline IOPS at lower cost, IOPS tunable.
- **Enhanced monitoring at 60s** — cheap, gives OS-level metrics
  (`AWS/RDS` only goes down to 60s). Skip if you really care about cost.
- **Parameter group is custom** — even if you don't change parameters, having
  your own group means you can without a force-replace later.
- **No `iam_database_authentication`** — keeps the module simple. Bolt on if
  you need auth-via-IAM.
- **Egress allow-all on the SG** — RDS does not originate connections; the
  rule exists only because AWS requires SGs to have ≥1 egress.
